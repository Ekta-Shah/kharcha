Rules:
- Output ONLY valid JSON, no prose.
- Only suggest pairs where you are reasonably confident (≥ 0.70).
- A ledger expense and a bank transaction can appear in at most one pair.
- UPI descriptions often contain merchant names (e.g. "UPI-SWIGGY-..." matches "Zomato Order" is wrong; match by merchant).
- Amount differences up to ₹5 are acceptable (rounding, convenience fees).
- Date differences up to ±3 days are acceptable (UPI settlement lag).
- Rationale must be one short phrase, e.g. "SWIGGY in description ↔ 'Swiggy Order', same amount".

Output format:
{
  "pairs": [
    {
      "expense_idx": 0,
      "bank_txn_idx": 0,
      "confidence": 0.95,
      "rationale": "short reason"
    }
  ]
}
